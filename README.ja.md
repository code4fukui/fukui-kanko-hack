# fukui-kanko-hack

店舗向けのiOS POS（販売時点情報管理）アプリ「ミケツ（Miketsu）」です。店舗スタッフがログイン後、支払い金額を入力し、顧客のQRコードをスキャンすることで決済処理を行うことができます。本アプリはSwift 5で開発されており、VIPERアーキテクチャとCombineフレームワークを採用し、バックエンドサービスとしてFirebaseを使用しています。

## 主な画面

- **ログイン:** 店舗スタッフがメールアドレスとパスワードを使用してログインするシンプルな画面です。
- **支払い:** ログイン後に表示される、支払い金額を入力するためのメイン画面です。
- **QRスキャナー:** 顧客のQRコードをスキャンするためのカメラビューです。QRコードには決済に必要な顧客のユーザー情報が含まれています。
- **確認:** 決済が成功すると、レジの音とともに成功アラートが表示されます。

## 機能

- **店舗認証:** Firebase Authenticationを使用した、店舗アカウントのセキュアなログインおよびログアウト。
- **QRコード決済:** ユーザーデータが含まれた顧客のQRコードをスキャンして決済を実行。
- **取引履歴の記録:** 各店舗の決済履歴を専用のFirestoreコレクションに記録。
- **モダンなアーキテクチャ:** 関心事を分離し保守性を高める、クリーンなVIPERアーキテクチャを採用。
- **リアクティブプログラミング:** 非同期イベントやデータフローの処理にCombineフレームワークを活用。

## 技術スタック

- **言語:** Swift 5
- **アーキテクチャ:** VIPER
- **フレームワーク:** UIKit, Combine, AVFoundation
- **バックエンド:** Firebase Authentication, Firebase Firestore
- **依存関係管理:** CocoaPods

## 要件

- Xcode 12 以降
- CocoaPods 1.11.3 以降

## セットアップ手順

1. **Firebaseのセットアップ:** [Firebase Console](https://console.firebase.google.com/)でプロジェクトを作成します。メール/パスワード認証を有効にし、Firestoreのセットアップを行ってください。
2. **設定:** Firebaseのプロジェクト設定から `GoogleService-Info.plist` ファイルをダウンロードし、`f-kanko/` ディレクトリ内に配置してください。
3. **依存関係のインストール:** ターミナルを開き、プロジェクトのルートディレクトリに移動して以下のコマンドを実行します:
    ```bash
    pod install
    ```
4. **ビルドと実行:** 生成された `f-kanko.xcworkspace` ファイルをXcodeで開き、プロジェクトを実行します。
5. **ログイン:** アプリにはデモ用にハードコードされたテスト用アカウントが含まれています。以下の情報を使用してログインしてください:
    - **メールアドレス:** `teststore@localhost.com`
    - **パスワード:** `teststore`
